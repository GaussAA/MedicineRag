# 医疗知识问答系统 - 项目需求文档

## 一、项目基本信息

### 1.1 项目概述

- **项目名称**: 基于RAG的医疗知识问答系统
- **项目类型**: Web应用
- **项目性质**: 本科毕业设计
- **核心目标**: 利用RAG技术,让用户能够用自然语言提问医疗健康问题,系统从医疗知识库中检索相关信息并生成准确回答
- **目标用户**:
  - 普通患者（健康咨询、常见病了解）
  - **用户技术水平**: 零技术背景

### 1.2 项目目标

- **核心目标**: 提供24小时可用的医疗咨询参考、减少重复性咨询工作
- **成功标准**: 回答准确率>85%、响应时间<10秒、用户满意度>4.0/5.0、单用户使用

## 二、数据集

### 2.1 知识库数据来源与版权说明

**数据来源**（推荐方案，优先选择有明确授权的）:

1. **开源医疗数据集**（优先使用，无版权风险）:
   - Chinese-Medical-Dialogue（中文医学对话数据集）- GitHub开源
   - cMedQA（中文医学问答数据集）- 学术数据集
   - 医患对话数据集（CSMDDC、IMCS-V2等）

2. **权威机构公开资料**（需遵守使用协议）:
   - WHO指南（可自由使用）
   - 国家卫健委发布的诊疗指南（政府公开信息）
   - 各医院公开的健康科普文章

3. **自有版权资料**（如适用）:
   - 购买或授权的专业医学教材
   - 内部培训资料

**推荐数据集**:

| 数据集名称               | 数据量     | 格式 | 版权        | 用途   |
| ------------------------ | ---------- | ---- | ----------- | ------ |
| Chinese-Medical-Dialogue | 约30万对话 | JSON | MIT开源协议 | 问答对 |
| cMedQA                   | 约10万问答 | JSON | 学术用途    | 问答对 |

**版权合规要求**:

- 使用开源数据集时遵守相应开源协议
- 不得直接复制受版权保护的完整书籍内容
- 可用于摘要、引用、生成独立创作
- 商业用途需确认数据许可协议

### 2.2 知识库规模

- **推荐数据量**: 约50MB-200MB（足够演示RAG效果）
- **知识库类型**: 通用医疗（常见病、用药指导、健康科普）
- **预估知识条目**: 500-2000个知识条目（如问答对、段落、疾病词条等）

> **说明**: 
> - 本科毕设不需要大规模数据，关键是展示RAG技术能力
> - 建议优先使用开源问答对数据集（如Chinese-Medical-Dialogue），数据量适中且质量有保证
> - 如使用PDF文档，50-100篇长文档即可覆盖常见医疗问题

### 2.3 问答场景

- **主要问题类型**:
  - 症状查询（"头痛是怎么回事？"）
  - 疾病知识（"糖尿病有哪些并发症？"）
  - 用药指导（"阿莫西林怎么服用？"）
  - 检查解读（"血常规报告怎么看？"）
  - 其他医疗相关的常见问题

   - 显示回答内容

### 3.2 特殊需求

**安全与合规**

#### 3.2.1 免责声明

- **展示位置**: 每次回答末尾必须显示
- **展示方式**: 固定文字 + 显著样式（红色/橙色提示框）
- **内容规范**:
  ```
  ⚠️ 免责声明：本回答仅供参考，不能替代医生诊断。如有严重症状，请立即就医。
  紧急情况请拨打120急救电话。
  ```

#### 3.2.2 敏感内容过滤

**需要过滤的敏感话题**:

| 类别     | 关键词示例       | 处理方式                   |
| -------- | ---------------- | -------------------------- |
| 自杀自残 | 自杀、割腕、服毒 | 拒绝回答，显示心理援助热线 |
| 暴力倾向 | 杀人、伤害他人   | 拒绝回答，建议报警         |
| 非法药物 | 毒品、吸毒、贩毒 | 拒绝回答，引导戒毒机构     |
| 色情内容 | 性交易、裸聊     | 拒绝回答                   |

**实现方案**:

- 部署本地敏感词库（如：dfkws-gfw或自建敏感词列表）
- 用户输入实时检测，敏感内容触发拦截提示
- 敏感问题统一引导至专业机构

#### 3.2.3 数据隐私

- **用户问题脱敏**: 去除姓名、身份证号、手机号等个人身份信息
- **存储策略**: 本地存储，不上传至第三方
- **日志管理**: 关闭详细请求日志或定期清理

## 四、技术架构

### 4.1 核心技术栈

```
应用前端：Streamlit（python原生，快速开发）
RAG 编排：LlamaIndex（专精rag）
向量数据库：Chroma（本地部署，开源免费）
LLM：Ollama qwen3:8b（本地模型，无需API费用）
Embedding：BGE-M3（Ollama本地模型，中文效果好）
包管理：uv（比起pip效率更高）
```

> **说明**: 
> - 本方案使用纯本地部署，无需API费用，完美契合学生预算
> - Ollama支持Windows/Mac/Linux，安装简单
> - 如果后续需使用API版本，可切换至OpenAI/DeepSeek API

### 4.2 技术参数配置

| 参数       | 推荐值            | 说明                                 |
| ---------- | ----------------- | ------------------------------------ |
| Chunk大小  | 512-1024字符      | 单个文档块包含的字符数               |
| Chunk重叠  | 50-100字符（10%） | 相邻块之间重叠区域，保持上下文连续性 |
| Top-K数量  | 3-5               | 初步检索的候选文档数量               |
| 相似度阈值 | 0.5               | 低于此分数的结果将被过滤             |
| 重排序启用 | 否（MVP阶段跳过） | 正式版启用二次排序提高相关性         |
| 温度参数   | 0.3               | LLM生成温度，建议低温度保持准确性    |
| 最大Token  | 1024              | 回答最大长度限制（MVP阶段适当限制）  |

### 4.3 开发环境

- **操作系统**: Windows 10/11
- **Python版本**: 3.10-3.12（推荐3.11）
- **GPU情况**: NVIDIA GeForce RTX 3050 Laptop GPU（8GB显存）
- **内存建议**: 16GB RAM

## 五、核心流程设计

### 5.1 知识库构建流程

```
上传文档 → 文档解析（提取文本）→ 文本分块（Chunk）
→ 向量化（Embedding）→ 存入向量数据库
```

**分块策略**:

- **方法**: 按语义切分（基于句子/段落边界）或固定长度切分
- **Chunk大小**: 512-1024字符
- **Chunk重叠**: 50-100字符（10%），避免信息断裂

### 5.2 问答流程

```
用户提问 → 问题向量化 → 向量检索（Top-K相似文档）
→ 构建Prompt（问题+检索内容）→ LLM生成回答 → 返回用户
```

**检索配置**:

- **检索数量（Top-K）**: 3-5个
- **相似度阈值**: 0.5（低于此分数不返回）
- **重排序（Rerank）**: MVP阶段跳过

### 5.3 典型使用场景

**场景1: 症状咨询**

- **用户输入**: "最近总是头晕，可能是什么原因？"
- **系统处理**:
  1. 检索知识库中关于"头晕"的医疗文档
  2. 提取相关段落（如：头晕的常见病因、伴随症状、就诊建议）
  3. LLM综合信息生成回答
- **预期输出**:
  ```
  头晕可能由多种原因引起，常见的包括：
  1. 低血压或贫血
  2. 颈椎问题
  3. 内耳前庭功能障碍
  ...
  建议您关注是否有其他伴随症状，如恶心、耳鸣等，并及时就医检查。

  参考来源:
  - 《常见症状鉴别诊断》第3章
  - 神经内科诊疗指南2023版
  
  ⚠️ 免责声明：本回答仅供参考，不能替代医生诊断。如有严重症状，请立即就医。
  ```

**场景2: 用药咨询**

- **用户输入**: "我有高血压，可以服用氨氯地平吗？有什么副作用？"
- **系统处理**:
  1. 检索知识库中关于"氨氯地平"的药物说明
  2. 提取适应症、禁忌症、副作用、药物相互作用等信息
  3. LLM综合生成安全用药建议
- **预期输出**:
  ```
  氨氯地平是一种常用的降压药，属于钙通道阻滞剂类。

  ✅ 适应症：原发性高血压、冠心病心绞痛
  ⚠️ 禁忌：对氨氯地平或其他二氢吡啶类药物过敏者禁用
  📋 常见副作用：头痛、踝部水肿、面部潮红、疲劳

  用药建议：
  1. 初始剂量通常为5mg，每日一次
  2. 建议在每天固定时间服用
  3. 服药期间定期监测血压
  4. 如与其他降压药合用，需遵医嘱调整剂量

  ⚠️ 重要提示：具体用药请遵医嘱，切勿自行调整剂量。
  ⚠️ 免责声明：本回答仅供参考，不能替代医生诊断。
  ```

**场景3: 检查报告解读**

- **用户输入**: "我的血常规检查显示白细胞偏高，是什么问题？"
- **系统处理**:
  1. 检索知识库中关于"血常规 白细胞升高"的医学知识
  2. 提取可能病因、临床意义、进一步检查建议
  3. LLM生成通俗易懂的解读报告
- **预期输出**:
  ```
  📊 血常规检查结果解读 - 白细胞计数偏高

  您的白细胞计数高于正常参考范围（正常值：4-10×10⁹/L）。

  🔍 可能的原因：
  1. 感染（最常见）：细菌感染、病毒感染
  2. 炎症反应：组织损伤、自身免疫性疾病
  3. 应激状态：手术、外伤、剧烈运动后
  4. 血液系统疾病：白血病、骨髓增殖性疾病（较少见）
  5. 药物影响：糖皮质激素可导致白细胞升高

  💡 建议：
  - 轻度假升高通常无需过度担心，可在一周后复查
  - 如伴随发热、乏力等症状，建议就医
  - 持续升高或明显异常需进一步检查

  ⚠️ 免责声明：本解读仅供参考，不能替代医生诊断。
  ```


### 6.2 数据管理

#### 6.2.1 知识库更新策略

| 更新类型 | 触发条件         | 实现方式                     |
| -------- | ---------------- | ---------------------------- |
| 增量更新 | 新增文档上传     | 实时解析并添加到向量数据库   |
| 全量重建 | 大规模知识库刷新 | 重新生成全量索引             |
| 删除更新 | 文档删除         | 软删除（标记删除）或重建索引 |

> **简化说明**: MVP阶段可直接删除向量库重新构建，无需复杂版本控制。

- **版本号格式**: v1.0.0（主版本.次版本.修订号）

#### 6.2.3 数据备份

- **备份频率**: 每周一次或重要更新前
- **备份位置**: 本地磁盘（项目目录外的位置）
- **简化说明**: 将Chroma向量数据库目录和原始文档目录打包备份即可

## 十、风险评估与应对措施

### 10.1 技术风险

| 风险                     | 等级 | 应对措施                         |
| ------------------------ | ---- | -------------------------------- |
| 本地GPU显存不足          | 中   | MVP阶段跳过rerank，使用轻量模型  |
| 模型首次加载慢           | 低   | 启动时预热模型，或限制模型大小   |
| 向量数据库性能瓶颈       | 低   | 数据量小，不会有问题             |
| 模型幻觉（生成错误信息） | 高   | 设置相似度阈值过滤，增加来源引用 |

### 10.2 业务风险

| 风险                     | 等级 | 应对措施                               |
| ------------------------ | ---- | -------------------------------------- |
| 回答错误导致用户延误治疗 | 高   | 强制显示免责声明，设置敏感问题引导就医 |
| 用户提问涉及医疗纠纷     | 中   | 记录对话日志（可选），保留证据链       |
| 知识库内容侵权           | 中   | 使用开源数据集，购买正版授权资料       |

### 10.3 安全风险

| 风险                  | 等级 | 应对措施                            |
| --------------------- | ---- | ----------------------------------- |
| 敏感内容（自杀/暴力） | 高   | 部署敏感词库，自动拦截+引导专业机构 |
| 用户数据泄露          | 高   | 本地存储，不上传第三方；数据脱敏    |
| 越狱提示词攻击        | 中   | 输入过滤+Prompt注入检测             |
