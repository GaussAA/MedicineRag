# 医疗知识问答系统 - 项目需求文档

## 一、项目基本信息

### 1.1 项目概述

- **项目名称**: 基于Agentic RAG技术的智能医疗知识问答系统
- **项目类型**: Web应用
- **项目性质**: 本科毕业设计
- **核心目标**: 利用Agentic RAG（智能体增强检索增强生成）技术，构建具备自主推理能力的医疗知识问答系统，实现更智能、更安全的医疗咨询体验
- **核心技术特色**: 
  - **ReAct推理模式**: 思考→行动→反思的智能决策循环
  - **多工具调用**: Agent自主选择检索、安全检查、追问等工具
  - **动态决策**: 根据检索结果自主判断是否需要补充检索
  - **知识缺口识别**: 自动识别知识库薄弱领域并主动追问
- **目标用户**:
  - 普通患者（健康咨询、常见病了解）
  - **用户技术水平**: 零技术背景

### 1.2 项目目标

- **核心目标**: 提供具备自主推理能力的24小时智能医疗咨询参考、减少重复性咨询工作、提升医疗知识获取效率
- **技术创新目标**: 展示Agentic RAG技术在医疗领域的应用，实现从被动检索到主动推理的升级
- **成功标准**: 回答准确率>85%、Agent推理成功率>90%、响应时间<10秒、用户满意度>4.0/5.0、单用户使用

### 3.2 特殊需求

**安全与合规**

#### 3.2.1 免责声明

- **展示位置**: 每次回答末尾必须显示
- **展示方式**: 固定文字 + 显著样式（红色/橙色提示框）
- **内容规范**:
  ```
  ⚠️ 免责声明：本回答仅供参考，不能替代医生诊断。如有严重症状，请立即就医。
  紧急情况请拨打120急救电话。
  ```

#### 3.2.2 敏感内容过滤

**需要过滤的敏感话题**:

| 类别     | 关键词示例       | 处理方式                   |
| -------- | ---------------- | -------------------------- |
| 自杀自残 | 自杀、割腕、服毒 | 拒绝回答，显示心理援助热线 |
| 暴力倾向 | 杀人、伤害他人   | 拒绝回答，建议报警         |
| 非法药物 | 毒品、吸毒、贩毒 | 拒绝回答，引导戒毒机构     |
| 色情内容 | 性交易、裸聊     | 拒绝回答                   |

**实现方案**:

- 部署本地敏感词库（如：dfkws-gfw或自建敏感词列表）
- 用户输入实时检测，敏感内容触发拦截提示
- 敏感问题统一引导至专业机构

#### 3.2.3 数据隐私

- **用户问题脱敏**: 去除姓名、身份证号、手机号等个人身份信息
- **存储策略**: 本地存储，不上传至第三方
- **日志管理**: 关闭详细请求日志或定期清理

### 4.2 Agentic RAG 技术架构

#### 4.2.1 Agent框架设计

**ReAct推理引擎**:
- **思考模块**: 问题理解、意图分析、上下文整合
- **决策模块**: 工具选择、执行顺序规划、资源分配
- **反思模块**: 结果评估、推理质量检查、继续决策
- **优化机制**: 单次LLM调用合并思考-决策-反思，提升效率

**工具体系**:
- **检索工具**: 智能文档检索，支持多轮迭代检索
- **安全检查工具**: 敏感内容检测、紧急症状识别
- **追问工具**: 基于问题和答案生成跟进问题
- **知识缺口工具**: 识别知识库薄弱领域并记录

**记忆管理**:
- **对话记忆**: 支持Token裁剪的上下文管理
- **推理历史**: 记录Agent推理过程用于分析优化
- **知识缺口库**: 收集用户咨询中的知识空白点

#### 4.2.2 推理流程优化

**单次调用优化**:
- 将传统ReAct的多次LLM调用优化为一次调用
- 通过精心设计的Prompt模板实现思考-决策-反思一体化
- 大幅提升响应速度，降低计算成本

**动态决策树**:
- 基于问题复杂度自动调整推理深度
- 支持条件分支：简单问题直接回答，复杂问题多轮推理
- 智能退出机制：达到置信度阈值或最大轮数时停止

**结果融合策略**:
- 多工具输出智能整合
- 置信度加权融合
- 矛盾信息冲突解决

## 五、核心流程设计

### 5.1 知识库构建流程

```
上传文档 → 文档解析（提取文本）→ 文本分块（Chunk）
→ 向量化（Embedding）→ 存入向量数据库
```

**分块策略**:

- **方法**: 按语义切分（基于句子/段落边界）或固定长度切分
- **Chunk大小**: 512-1024字符
- **Chunk重叠**: 50-100字符（10%），避免信息断裂

### 5.2 Agentic RAG 问答流程

```
用户问题 → Agent接收问题 → 进入ReAct推理循环
           ├── 步骤N: 思考+决策+反思（一次LLM调用）
           │         └── 分析意图 → 选择工具 → 判断是否继续
           ├── 执行行动：调用工具（检索/安全检查/追问/知识缺口）
           │         └── 并行或顺序执行多个工具
           ├── 观察结果 → 更新上下文
           └── 生成最终回答（含推理过程展示）
```

**核心优化点**:
- **单次LLM调用**: 将思考-决策-反思合并为一次调用
- **动态工具选择**: 根据问题复杂度自主选择工具组合
- **智能退出**: 达到置信度阈值或最大轮数时停止推理
- **结果融合**: 多工具输出智能整合

### 5.3 Agentic RAG 核心特性

#### 5.3.1 ReAct推理模式
- **思考阶段**: LLM分析用户问题意图，理解上下文
- **决策阶段**: 基于思考结果选择合适的工具组合
- **反思阶段**: 评估工具执行结果，判断是否需要继续推理
- **优化策略**: 合并思考-决策-反思为单次LLM调用，提升响应速度

#### 5.3.2 动态工具调用
- **自主决策**: Agent根据问题复杂度自主决定调用哪些工具
- **多轮推理**: 支持最多3轮推理循环，避免过度推理
- **工具组合**: 可同时调用多个工具（如安全检查+检索）
- **结果融合**: 智能整合多个工具的输出结果

#### 5.3.3 智能对话增强
- **上下文感知**: 自动获取最近3轮对话历史
- **主动追问**: 识别问题信息不足时主动生成跟进问题
- **知识缺口识别**: 发现知识库薄弱领域并记录
- **Token管理**: 自动裁剪对话历史，避免上下文溢出

#### 5.3.4 推理过程展示
- **步骤可视化**: 前端展示Agent的推理步骤和工具调用过程
- **透明化**: 用户可查看Agent的思考过程和决策依据
- **可解释性**: 提供推理过程解释，提升用户信任

**场景1: 复杂症状咨询（Agentic RAG优势）**

- **用户输入**: "最近总是头晕，伴随恶心和耳鸣，持续了三天，该怎么办？"
- **Agent推理过程**:
  1. **思考**: 分析症状组合（头晕+恶心+耳鸣），判断为前庭系统相关问题
  2. **安全检查**: 确认无紧急症状，无需立即就医建议
  3. **检索工具**: 调用检索获取前庭神经炎相关知识
  4. **反思**: 检索结果充分，信息完整，无需补充检索
  5. **追问工具**: 生成跟进问题："症状是否在转头时加重？"
- **预期输出**:
  ```
  🤖 Agent推理过程：
  步骤1: 分析症状组合 → 初步判断为前庭系统问题
  步骤2: 安全检查 → 无紧急情况
  步骤3: 检索相关知识 → 获取前庭神经炎信息
  步骤4: 评估结果 → 信息充分
  步骤5: 生成追问 → "症状是否在转头时加重？"

  💊 症状分析：
  您的症状（头晕、恶心、耳鸣）可能与前庭神经炎有关...

  ❓ 跟进问题：症状是否在转头或改变体位时加重？这有助于进一步诊断。

  参考来源: 《神经内科诊疗指南2023版》
  ⚠️ 免责声明：本回答仅供参考，不能替代医生诊断。
  ```

**场景2: 知识缺口识别（Agentic RAG特色）**

- **用户输入**: "我可以吃中药调理甲状腺功能吗？"
- **Agent推理过程**:
  1. **思考**: 识别问题涉及中西医结合治疗
  2. **检索工具**: 查询甲状腺相关知识
  3. **反思**: 发现知识库中西药信息丰富，中药信息不足
  4. **知识缺口工具**: 记录"中药治疗甲状腺疾病"知识缺口
  5. **回答生成**: 基于现有知识回答，并提示知识局限性
- **预期输出**:
  ```
  🤖 Agent推理过程：
  步骤1: 问题分析 → 中西医结合治疗咨询
  步骤2: 知识检索 → 获取甲状腺基础知识
  步骤3: 结果评估 → 中药信息不足
  步骤4: 知识缺口识别 → 记录中药治疗领域缺失

  💊 回答：
  关于中药调理甲状腺功能，目前知识库中相关信息有限...

  📝 知识库建议：建议补充中医治疗甲状腺疾病的相关资料

  ⚠️ 免责声明：本回答基于现有知识库，不能替代专业医疗建议。
  ```

**场景2: 用药咨询**

- **用户输入**: "我有高血压，可以服用氨氯地平吗？有什么副作用？"
- **系统处理**:
  1. 检索知识库中关于"氨氯地平"的药物说明
  2. 提取适应症、禁忌症、副作用、药物相互作用等信息
  3. LLM综合生成安全用药建议
- **预期输出**:
  ```
  氨氯地平是一种常用的降压药，属于钙通道阻滞剂类。

  ✅ 适应症：原发性高血压、冠心病心绞痛
  ⚠️ 禁忌：对氨氯地平或其他二氢吡啶类药物过敏者禁用
  📋 常见副作用：头痛、踝部水肿、面部潮红、疲劳

  用药建议：
  1. 初始剂量通常为5mg，每日一次
  2. 建议在每天固定时间服用
  3. 服药期间定期监测血压
  4. 如与其他降压药合用，需遵医嘱调整剂量

  ⚠️ 重要提示：具体用药请遵医嘱，切勿自行调整剂量。
  ⚠️ 免责声明：本回答仅供参考，不能替代医生诊断。
  ```

**场景3: 检查报告解读**

- **用户输入**: "我的血常规检查显示白细胞偏高，是什么问题？"
- **系统处理**:
  1. 检索知识库中关于"血常规 白细胞升高"的医学知识
  2. 提取可能病因、临床意义、进一步检查建议
  3. LLM生成通俗易懂的解读报告
- **预期输出**:
  ```
  📊 血常规检查结果解读 - 白细胞计数偏高

  您的白细胞计数高于正常参考范围（正常值：4-10×10⁹/L）。

  🔍 可能的原因：
  1. 感染（最常见）：细菌感染、病毒感染
  2. 炎症反应：组织损伤、自身免疫性疾病
  3. 应激状态：手术、外伤、剧烈运动后
  4. 血液系统疾病：白血病、骨髓增殖性疾病（较少见）
  5. 药物影响：糖皮质激素可导致白细胞升高

  💡 建议：
  - 轻度假升高通常无需过度担心，可在一周后复查
  - 如伴随发热、乏力等症状，建议就医
  - 持续升高或明显异常需进一步检查

  ⚠️ 免责声明：本解读仅供参考，不能替代医生诊断。
  ```

| -------- | ---------------- | ---------------------------- |
| 增量更新 | 新增文档上传     | 实时解析并添加到向量数据库   |
| 全量重建 | 大规模知识库刷新 | 重新生成全量索引             |
| 删除更新 | 文档删除         | 软删除（标记删除）或重建索引 |

> **简化说明**: MVP阶段可直接删除向量库重新构建，无需复杂版本控制。

#### 6.2.2 版本控制（可选）

- **版本号格式**: v1.0.0（主版本.次版本.修订号）
- **简化方案**: 每次重要更新记录文档列表即可，无需复杂回滚机制

#### 6.2.3 数据备份

- **备份频率**: 每周一次或重要更新前
- **备份位置**: 本地磁盘（项目目录外的位置）
- **简化说明**: 将Chroma向量数据库目录和原始文档目录打包备份即可

## 七、性能与约束

### 7.1 性能指标

| 指标            | Pipeline RAG目标 | Agentic RAG目标 | 说明                        |
| --------------- | ---------------- | --------------- | --------------------------- |
| 响应时间        | <10秒            | <12秒           | Agent推理增加少量开销       |
| Agent推理成功率 | -                | >90%            | Agent正确完成推理任务的比例 |
| 平均推理轮数    | -                | 1.5-2.5轮       | 平衡效率和准确性            |
| 并发用户        | 1人              | 1人             | 本地部署，单用户使用        |
| 准确性要求      | >85%             | >88%            | Agent推理提升准确性         |
| 可用性          | 99.9%            | 99.9%           | 系统正常运行时间比例        |

**Agentic RAG性能说明**:
- 推理时间增加2-3秒，但准确性显著提升

- 支持最大3轮推理，避免过度计算

### 7.2 约束条件

- **预算限制**: 0元（纯本地部署，无API费用）
- **硬件限制**: 个人笔记本电脑（RTX 3050 8GB显存）
- **合规要求**: 不涉及真实患者数据，仅使用公开科普知识

## 十、风险评估与应对措施

### 10.1 技术风险

| 知识库内容侵权           | 中   | 使用开源数据集，购买正版授权资料       |

| 风险                  | 等级 | 应对措施                            |
| --------------------- | ---- | ----------------------------------- |

| 用户数据泄露          | 高   | 本地存储，不上传第三方；数据脱敏    |
| 越狱提示词攻击        | 中   | 输入过滤+Prompt注入检测             |
